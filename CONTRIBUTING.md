# 贡献指南

欢迎补充经过真实任务验证的提示词。这个仓库更看重“可以稳定完成任务”，而不是提示词的长度。

## 新增提示词

1. 复制 `templates/prompt-template.md`。
2. 先选择最贴切的大类；没有合适大类时，创建一个小写英文、以连字符分隔的新分类目录，并补充该分类的 `README.md`。
3. 在 `prompts/<分类>/` 下创建一个小写英文、以连字符分隔的提示词目录，例如 `academic-research/pdf-review`。
4. 将可复制内容保存为与目录同名的 `.md` 文件（如 `pdf-review.md`），补充说明保存为 `README.md`。
5. 在分类 README、`prompts/README.md` 和仓库根目录的 `README.md` 中增加入口。

## 提交前检查

- 任务目标、输入和交付物是否明确。
- 是否区分只读检查、可逆操作和不可逆操作。
- 是否为删除、覆盖、付款、发布、发信等高影响操作设置了清晰边界。
- 是否写明完成后的验证方法。
- 是否移除了真实姓名、账号、令牌、私人绝对路径和其他敏感信息。
- 是否避免依赖某一次对话中未写进提示词的背景信息。
- Markdown 链接和代码块是否可以正常显示。
- 运行 `python3 scripts/validate_docs.py` 并确认通过。

## 写作约定

- 文件名使用小写英文和连字符。
- 面向使用者的说明以中文为主，必要时补充英文术语。
- 使用占位符表达可变信息，例如 `<目标路径>`、`<仓库地址>`。
- 对危险操作使用明确的“允许”和“禁止”，不要只写“请谨慎”。
- 提示词应让执行者在结束时给出简洁、可核对的结果摘要。
