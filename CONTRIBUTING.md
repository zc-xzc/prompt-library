# 贡献指南

欢迎补充经过真实任务验证的提示词。这个仓库更看重“可以稳定完成任务”，而不是提示词的长度。

## 新增提示词

1. 从 	emplates/prompt-template.md 复制模板。
2. 在 prompts/ 下创建一个小写英文、连字符分隔的目录，例如 pdf-review/。
3. 将提示词保存为 <目录名>.md（如 pdf-review.md），使用说明保存为 README.md。
4. 在 prompts/README.md 和仓库根 README.md 中增加入口。

## 提交前检查

- [ ] 任务目标、输入和交付物是否明确
- [ ] 是否区分只读检查、可逆操作和不可逆操作
- [ ] 是否为删除、覆盖、付款、发布、发信等高影响操作设置了清晰边界
- [ ] 是否写明完成后的验证方法
- [ ] 是否移除了真实姓名、账号、令牌、私人绝对路径和其他敏感信息
- [ ] 是否避免依赖某一次对话中未写进提示词的背景信息
- [ ] Markdown 链接和代码块是否可以正常显示

## 写作约定

- 文件名和目录名使用小写英文 + 连字符（-，kebab-case），如 macos-cleanup.md，不使用下划线
- 面向使用者的说明以中文为主，必要时补充英文术语
- 使用占位符表达可变信息，例如 <目标路径>、<仓库地址>
- 对危险操作使用明确的“允许”和“禁止”，不要只写“请谨慎”
- 提示词应让执行者在结束时给出简洁、可核对的结果摘要

## PR 流程

1. Fork 本仓库并创建功能分支
2. 完成修改后发起 Pull Request 到 main 分支
3. PR 标题使用中文，描述变更内容和验证情况
4. 至少需要一次审查通过后才能合并
5. 合并后删除功能分支
