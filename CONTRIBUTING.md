# 贡献指南

欢迎补充经过真实任务验证的技能和提示词。

## 新增内容

### 技能

1. 确定所属领域和分类（若无合适分类，创建新的并补充 `README.md`）
2. 在 `skills/<领域>/<分类>/` 下创建技能目录（小写英文 + 连字符）
3. 技能必须包含 `SKILL.md`
4. 补充 `README.md` 说明使用场景和注意事项
5. 在分类 `README.md` 中增加入口

### 提示词

1. 复制 `templates/prompt-template.md` 作为起点
2. 在对应分类下创建目录（命名与提示词文件名一致）
3. 保存为 `.md` 文件，补充 `README.md`
4. 在分类 `README.md` 中增加入口

## 提交前检查

- 任务目标、输入和交付物是否明确
- 是否区分只读检查、可逆操作和不可逆操作
- 是否为高影响操作设置清晰边界
- 是否写明验证方法
- 是否移除敏感信息
- 不提交生成的依赖、构建产物或本地环境文件（例如 `node_modules/`、`.venv/`、`dist/`）
- 运行 `python scripts/validate_docs.py` 并确认通过

## 写作约定

- 文件名使用小写英文和连字符
- 面向使用者的说明以中文为主
- 使用占位符表达可变信息，例如 `<目标路径>`、`<仓库地址>`
- 对危险操作使用明确的"允许"和"禁止"
