# macOS 软件缓存与残留清理

这是一份面向 Codex 等本机智能体的完整提示词。复制下面整个代码块即可使用。

## 可直接使用的提示词

```text
请在我的 macOS 上完成一次保守、可验证的软件缓存与残留清理。

【本次参数】
- 主要检查范围：~/Library/Containers、~/Library/Caches、~/Library/Logs
- 重点软件：微信、WPS、腾讯会议、网易云音乐、汽水音乐、Microsoft Edge、O+Connect、PyCharm、EasyConnect
- 允许正常退出正在运行的目标应用：是
- 允许永久删除已经确认可再生的缓存、日志和临时文件：是
- 是否清空废纸篓：否

【目标】
1. 找出占用空间最大的应用目录和可安全清理项。
2. 清理高置信度的缓存、日志、临时文件、崩溃记录和已经完成安装的旧升级包。
3. 保护聊天记录、附件、文档、账号信息、配置、插件、运行组件和云端/离线数据。
4. 在结束时报告实际释放空间、仍然较大的目录和未处理原因。

【执行方式】
第一阶段只读检查：
1. 记录清理前的磁盘可用空间。
2. 检查上述三个范围的总大小，并列出最大的应用目录与子目录；不要只根据目录名称判断用途。
3. 检查重点软件是否正在运行，并识别可能存在未保存工作的应用。
4. 将候选项分为三类：
   - 高置信度可清理：明确的 Cache/Caches、Logs、tmp/temp、崩溃记录、缩略图缓存、浏览器或媒体缓存、已经完成安装且不再需要的旧升级包。
   - 需要保留：用户文档、聊天数据库与附件、下载内容、账号与登录信息、应用配置、插件、字体/模板资源、云盘或离线文件。
   - 无法确认：用途不明、名称模糊、内容混合或可能影响应用功能的目录。

第二阶段安全清理：
5. 只处理“高置信度可清理”项目。执行前先在工作记录中列出每个精确路径、当前大小和判断依据，然后按本提示词授权继续，不必重复向我确认。
6. 如果相关应用正在运行：
   - 对没有未保存工作风险的后台程序，可正常退出后再清理。
   - 对可能包含未保存文档、会议、下载或同步任务的应用，不要强制结束；跳过相关项并在结果中说明。
7. 每次只删除已经解析并核对过的精确路径。不得使用指向整个用户目录、整个 Library、整个 Containers、整个应用容器或其他宽泛位置的递归删除。
8. 不得依赖未解析变量、模糊匹配或宽泛通配符决定删除目标；不得跟随符号链接删除目标之外的内容。
9. 优先使用可恢复方式。只有在本次参数明确允许，且目标确认为可自动重建的缓存、日志或临时文件时，才可精确永久删除。
10. 如应用依赖某个缓存目录本身，可在清理后重建空目录并保持合理权限。

【绝对禁止】
- 不删除任何完整的应用容器。
- 不删除微信或其他通讯软件的聊天记录、附件、数据库、账号数据和用户主动保存的文件。
- 不删除 WPS 或其他办公软件的文档、账号数据、用户模板、插件和正常运行组件。
- 不删除云盘同步数据、离线文件或同步状态数据库。
- 不删除 macOS 核心系统目录、系统服务数据或用途不明的大目录。
- 不为了追求释放空间而清理“需要保留”或“无法确认”的项目。
- 不清空废纸篓，除非本次参数明确改为“是”。

【针对重点软件的判断提示】
- 微信：缓存、临时文件和日志可以评估；聊天数据、xwechat_files 中的用户内容及数据库必须保留。
- WPS：升级完成后可评估旧升级包、日志和缓存；office6、插件、字体、模板及用户数据用途不明确时保留。
- 会议、音乐和浏览器软件：优先处理媒体缓存、网页缓存、日志和旧升级包；保留下载文件、收藏、登录数据及用户配置。
- 开发工具和企业连接工具：只清理明确日志和缓存；保留项目、证书、连接配置、插件和运行环境。

【完成后的验证】
1. 重新统计已处理路径，确认目标已清理或缩小。
2. 记录清理后的磁盘可用空间，并计算实际释放量。
3. 再次列出检查范围中占用最大的目录，区分“用户数据，建议在应用内管理”和“以后仍可定期清理”。
4. 检查是否有应用因无法安全退出而被跳过。
5. 用中文给出最终摘要，至少包括：
   - 已清理的软件与项目；
   - 每项释放空间和合计释放空间；
   - 明确保留的关键数据；
   - 跳过项及原因；
   - 当前磁盘可用空间；
   - 下次建议清理时间或触发条件。

整个过程保持保守：不确定就保留并说明，不以“看起来像缓存”作为唯一删除依据。
```
