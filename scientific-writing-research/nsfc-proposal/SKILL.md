# 国家自然科学基金（NSFC）申请书撰写

## 概述

本 Skill 指导 AI Agent 协助用户撰写**国家自然科学基金**申请书，依据2026年度官方最新版规范，覆盖青年（C类）、优青（B类）、杰青（A类）三类项目。区别于本仓库已有的 `research-grants`（面向 NSF/NIH/DOE 等海外基金），本 Skill 专为**中国大陆 NSFC 体系**设计，遵循"瘦身提质"改版后的三大正文模块结构（立项依据 / 研究内容 / 研究基础），并对2026年新评审机制（分阶段评审、前500字初筛、AI 辅助声明）做了重点提示。

**学术诚信声明**：禁止使用 AI 直接生成申请书内容；AI 仅可用于辅助检索文献、整理结构、润色语言。所有科学观点、研究思路、创新点必须由申请人本人提供，并由人工核实事实准确性。

---

## 何时使用本 Skill

当用户的请求涉及以下情形时应触发本 Skill：

- 撰写、修改或润色 NSFC 申请书的任一模块
- 询问青基 / 优青 / 杰青的申报规范、字数限制、申请代码、亚类说明
- 检查申请书是否符合 2026 年新版"瘦身"要求（不超过30页、无独立"研究方案"板块等）
- 评估立项依据是否在前500字内呈现创新性（应对分阶段评审）
- 整理 5 篇代表作、撰写"本人贡献声明"
- 撰写伦理审查、AI辅助声明、人类遗传资源说明等合规文档

## 三类项目核心差异速查

| 维度 | C类（青基） | B类（优青） | A类（杰青） |
|---|---|---|---|
| 申请人年龄 | 男≤35 / 女≤40 | 男≤35 / 女≤40 | 男≤45 / 女≤48 |
| 资助金额 | ~24~30万 | ~200万 | ~400万（延续800万） |
| 资助期限 | 3年 | 3年 | 5年 |
| 评审焦点 | 创新潜力 | 已有较好研究成绩 | 顶尖人才已取得突出成绩 |
| 正文上限 | 不超过30页 | 不超过30页 | 不超过30页 |
| 经费 | 包干制，无需编预算 | 包干制 | 包干制 |
| 申请书结构（2026） | 立项依据+研究内容+研究基础 | 同左 | 同左 + 申请人情况 |

> 详细规范见 `references/NSFC_WRITING_SPEC.md`。

## 工作流程

### Phase 1 · 信息收集

通过对话（或在支持的工具中使用结构化提问）收集以下要素：

```
【申请类型】：C类（青基）/ B类（优青）/ A类（杰青）/ A类延续
【项目名称】：20字以内名词性短语
【申请代码】：必须选至二级（4位数字），可附2~3个辅助代码
【研究期限】：C/B类3年；A类5年
【关键词】：3-5个，含研究对象、核心方法、关键科学问题
【申请人信息】：职称、单位、研究方向、年龄（核对资格）
【拟解决的核心科学问题】：1-2句话
【现有研究基础】：已发论文（按相关性排序）、初步实验数据、技术平台
【是否涉及伦理 / 人遗 / 生物安全 / 临床】：用于决定附件清单
【是否使用 AI 辅助检索】：用于撰写AI使用声明
```

如果用户信息不全，针对最影响 Agent 决策的字段（**申请类型、核心科学问题、研究基础**）追问；其余可填"请自行补充"占位。

### Phase 2 · 文献检索

参照 `references/LITERATURE_SEARCH.md`，按学科领域选择检索策略：

- **如果已安装 `qinyan-paper-search` Skill** → 优先调用其统一检索接口（覆盖 Google Scholar / PubMed / ArXiv / 万方）
- **如果可用 `WebSearch` / `WebFetch`** → 用关键词组合检索近5年综述与代表性论文
- **如果未配置外网工具** → 提示用户在本地准备 PDF/Markdown 文献库，Agent 据此撰写综述

**学科取向建议**：

| 学部方向 | 优先检索库 | 备注 |
|---|---|---|
| 生命科学 / 医学 | PubMed → Web of Science → 万方 | PubMed 仅英文 query |
| 化学 / 材料 | Web of Science → SciFinder → ArXiv | |
| 信息 / 数学 / 物理 | ArXiv → Google Scholar → DBLP | |
| 工程 / 环境 | Web of Science → Google Scholar → 万方/CNKI | |
| 交叉学科 | OpenAlex → Google Scholar | 用 OpenAlex 获取引用网络 |

### Phase 3 · 立项依据写作（6~10页）

按 `references/NSFC_WRITING_SPEC.md` §3.1 的"倒金字塔"结构：

1. **开篇 500 字**（极为关键，2026年分阶段评审初筛重点）：
   - 第①句：本研究的核心科学问题与原创性切入点
   - 第②段：与现有研究的本质区别
   - 第③段：潜在科学价值
   - **严禁**开头大段堆砌常规综述
2. **研究背景与国内外现状**：宏观 → 中观 → 微观空白，覆盖近5年核心文献
3. **科学问题与立项依据**：明确"why/how"型基础科学问题（非工程"how to do"）
4. **C类 vs B/A类区别**：
   - C类侧重申请人**独立提出问题的能力**
   - B/A类需充分前期工作支撑

### Phase 4 · 研究内容 + 研究基础写作

**研究内容**（10~14页）— 2026 年改革要求：

- 取消"研究方案 / 技术路线 / 年度计划"独立板块的强制要求
- 研究方法须**自然融入**研究内容叙述
- 各子内容必须统一在**核心科学问题**这条轴线上，逻辑递进而非平行罗列
- 特色与创新点建议单独成段，2~4 条，每条含 [类型]+[现状]+[突破]+[意义] 四要素

**研究基础**（6~10页）— 5 篇代表作选排黄金法则：

| 维度 | 要求 |
|---|---|
| 时效性 | 近5年优先；CNS正刊/高水平子刊可放宽 |
| 期刊水平 | 至少3篇二区及以上；避开预警/On-hold 期刊 |
| 作者顺序 | C类侧重一作（≥2篇唯一一作二区以上） |
| 相关性 | 5篇围绕同一方向，形成"机制发现→验证→拓展"链条 |
| 理想配置 | 3篇直接相关 + 1篇拓展 + 1篇高影响力旁证 |

**代表作个人贡献声明格式**（2026 强化要求）：
> "本人在该研究中负责：[具体贡献]，约占工作量的 [X]%。"

### Phase 5 · 合规与自检

完成草稿后，逐项核对 `references/QUALITY_CHECKLIST.md` 中的：

- **形式完整性**：申请代码、页数、附件（伦理/人遗/生物安全）
- **内容质量**：开篇500字、创新点是否有文献对比支撑、代表作相关性
- **诚信合规**：AI 辅助声明、超项核查、与在研项目区别

输出最终申请书草稿（Markdown 格式），并附自检结果摘要。

## 高频错误警示

| 错误 | 改正 |
|---|---|
| 套话堆砌（"具有重大理论意义"） | 用具体数据、机制、对比说明意义 |
| 研究方向发散（≥5个独立子方向） | 聚焦 2-3 个核心问题深挖 |
| 目标虚大（"建立完整理论体系"） | 改为 3 年内可实现的具体科学目标 |
| 方法缺失 | 在研究内容中融入关键方法说明 |
| 创新点无文献对比支撑 | 每条配"现状+突破"对比说明 |
| 篇幅失控（超过30页） | 删除套话，每句话须有实质信息量 |
| 代表作与课题无关 | 只选强相关成果，详细说明个人贡献 |

> **最终提醒**：申请书须由申请人本人撰写。本 Skill 仅作为结构框架与写作指导，所有科学内容必须来自申请人本人的研究工作与学术思考。高质量的申请书体现的是申请人的学术思维与表达能力，而非对模板的套用。
