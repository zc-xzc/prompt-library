# 国家自然科学基金青年科学基金申报书撰写规范
## Agent 写作指导 Prompt（2026年度最新版）

---

## 〇、使用前说明：三类项目差异速查

在开始撰写前，必须明确申请的是哪一类项目，三类在评审侧重、篇幅、申请资格上存在本质差异：

| 维度 | **C类（青基）** | **B类（优青）** | **A类（杰青）** |
|---|---|---|---|
| 原名称 | 青年科学基金 | 优秀青年科学基金 | 国家杰出青年科学基金 |
| 申请人年龄 | 男≤35岁 / 女≤40岁 | 男≤35岁 / 女≤40岁 | 男≤45岁 / 女≤48岁 |
| 资助金额 | ~24~30万元 | ~200万元 | ~400万元（延续800万）|
| 资助期限 | 3年 | 3年 | 5年 |
| **核心评审焦点** | **申请人创新潜力** | **已取得较好研究成绩** | **已取得突出成绩的顶尖人才** |
| 正文篇幅上限 | **原则上不超过30页** | 不超过30页 | 不超过30页 |
| 经费预算 | **包干制，无需编制预算** | 包干制 | 包干制 |
| 申请书结构（2026版） | 立项依据 + 研究内容 + 研究基础 | 立项依据 + 研究内容 + 研究基础 + 申请人情况 | 同左，另需附延续资助材料 |

> **⚠️ 重要提示（2026年改版）**：面上项目与青年C类申请书已于2026年完成"瘦身提质"改版，取消了"研究方案""技术路线""年度计划"等独立板块的强制要求，正文仅保留三个核心模块。本规范以2026年最新版为准。

## 一、基本信息填写规范

### 1.1 必填字段规范

```
项目名称：
  - 20字以内，名词性短语，精准反映核心科学问题
  - 避免使用"研究""探究""关于"等冗余词汇
  - 格式参考：[研究对象/系统] + [核心机制/效应] + [研究方法/角度]（可选）
  - 示例：铁基超导体自旋-轨道耦合驱动拓扑相变机制

资助类别：青年科学基金项目
亚类说明：（C类 / B类 / A类 / A类延续资助，按实填写）

申请代码：
  - 尽量选至二级申请代码（4位数字）
  - 生命科学部等科学部强制要求选至二级代码，否则不予受理
  - 可选2-3个辅助代码辅助分流评审

研究期限：3年（C类/B类） / 5年（A类）
摘要：不超过400字（见第二章详细规范）
关键词：3-5个，涵盖研究对象、核心方法、关键科学问题
```

### 1.2 不予受理的常见形式错误（必须规避）

- 未按要求提供伦理委员会证明、推荐信、导师同意函等附件材料
- 申请代码选择错误或仅选至一级代码
- 研究期限填写错误（C类/B类必须填3年）
- 申请人或参与者存在超项情况
- 申请书缺页、页数超限
- 主要参与者个人简历未在线填写并上传PDF

## 二、摘要撰写规范

**字数**：不超过400字  
**语气**：陈述式、肯定式，避免"拟""将""可能"等不确定表述  
**结构**（建议顺序）：

```
① 研究背景与问题切入（1-2句）：点明领域重要性，引出核心科学问题
② 现有研究的不足/空白（1句）：精准指出"gap"，与本研究形成对比
③ 研究内容（1-2句）：说明"做什么"，聚焦核心不展开细节
④ 研究方法/手段（1句）：关键技术或独特方法路径
⑤ 预期成果与科学意义（1-2句）：成果形式 + 对领域的贡献
```

**示例框架**：
> "……领域中，[X问题]至关重要。然而，现有研究尚未阐明[Y机制]。本项目拟以[研究对象]为模型，通过[核心方法]，系统研究[核心内容]。预期揭示[科学发现]，为[领域应用/理论发展]提供重要理论依据。"

## 三、正文撰写规范（2026版三大模块）

> **总体原则**：正文原则上不超过30页，鼓励简洁表达。摒弃"套路化""八股文"写法，回归科学研究本质。禁止使用AI直接生成申请书内容；如用AI辅助检索文献，须人工核实所有信息准确性并全面声明使用情况。

---

### 3.1 第一模块：立项依据

**核心任务**：说清楚"**为什么要开展此项研究，研究的价值何在**"

**篇幅建议**：6~10页

**【2026年评审新机制警示】**  
分阶段评审机制下，立项依据的**前500字**可能是初筛阶段评审专家重点阅读的内容，直接决定是否进入深度评审。必须在开头直击要害，采用"倒金字塔"结构，开门见山呈现创新性。

#### 3.1.1 开篇（前500字，极为关键）

必须在开篇清晰回答以下三个问题：

```
① 研究问题的原创性：这是一个"新问题"还是对已知问题的新视角？
② 与现有研究的本质区别：不是简单补充，而是有何根本突破？
③ 潜在科学价值：一旦成功，能改变什么认知或解决什么难题？
```

**禁止**在开头大篇幅堆砌常规文献综述；**推荐**直接从"发现的科学空白"或"现有认知的核心矛盾"切入。

#### 3.1.2 研究背景与国内外研究现状

撰写要点：
- 按"宏观背景 → 中观研究脉络 → 微观研究空白"逐层聚焦
- 文献综述须客观、全面、精准，覆盖近5年核心文献
- 指出现有研究的**具体局限性**（非泛泛而谈），并与本研究形成直接对照
- 语言：不能只堆砌他人工作，需有自己的分析与判断

格式要求：
- 参考文献须规范引用（建议APA或期刊指定格式）
- 图表引用须注明出处，并保证版权合规

#### 3.1.3 科学问题与立项依据

```
明确提出：本项目拟解决的核心科学问题是什么？
   → 科学问题应是基础研究层面的"why"或"how"，而非工程技术层面的"how to do"

说明科学价值：
   → 理论意义：对本领域知识体系的推进
   → 应用前景：可能引发的下游应用（可选，不宜过度强调）
```

**特别注意（C类 vs B/A类区别）**：
- **C类**：立项依据应着重体现申请人**独立提出问题的能力**，展示独特的科学视角，不必过分依赖前期大量积累来"撑场面"，创新潜力比已有成果更重要
- **B/A类**：立项依据需有充分的前期工作支撑，体现研究的系统性与深度

### 3.2 第二模块：研究内容

**核心任务**：说清楚"**做什么、怎么做、为什么这样做**"

**篇幅建议**：10~14页

**2026年改革重点**：本模块**不再预设提纲限制**，申请人须根据自己的研究思路和研究工作的内在逻辑自主撰写，禁止套用固化模板。

#### 3.2.1 研究内容的组织原则

```
核心轴线：所有研究内容必须统一在"核心科学问题"这条轴线上
层次关系：各子内容之间应有清晰的逻辑递进关系，而非平行罗列
整体性：内容、目标、方法三者须一体化，互相印证
```

研究内容组织的四个基本原则：
1. **紧扣科学问题**：每个子问题都能对应核心科学问题的某个维度
2. **适度集中**：宁可深入挖掘少量核心问题，不要面面俱到、广而不深
3. **逻辑自洽**：各研究内容间的因果/递进关系清晰可读
4. **可验证性**：每项内容都应有明确的验证手段或预期产出

#### 3.2.2 研究目标

```
撰写要求：
  - 明确、具体、可量化（或可评估）
  - 与研究内容一一对应，避免目标比内容"大"
  - 区分"科学目标"（理解什么）和"技术目标"（实现什么）
  - 3年内可实现，避免过于宏大
```

**示例**：
> 本项目的科学目标是：阐明[X系统]中[Y效应]的分子机制，建立[Z]的理论模型，为……提供科学依据。

#### 3.2.3 研究方法与可行性（融入研究内容中阐述）

2026年改版取消了独立的"研究方案"板块，但研究方法必须在研究内容的叙述中**自然融入**：

```
对于关键研究内容：
  → 说明采用的核心方法/技术手段
  → 说明选择该方法的理由（相比其他方法的优势）
  → 预判可能的技术难点及解决思路
  → 可附关键技术路线图（建议保留，1-2张即可）
```

可行性论证要点：
- 申请人是否具备相关技术能力（对应研究基础部分）
- 实验条件是否具备
- 如有创新性方法，需额外说明初步验证情况

#### 3.2.4 特色与创新点

**建议单独呈现**，条目化列出（2-4条为宜），层次分明：

```
每条创新点的撰写框架：
  [类型标注]：学术思想创新 / 研究内容创新 / 研究方法创新
  [现状描述]：以往研究做了什么、局限在哪里
  [本研究突破]：本研究如何突破，具体新在哪里
  [科学意义]：这一创新对领域的潜在影响
```

三类创新的写法导向：
- **学术思想创新**：提出新假说、新理论框架、新研究范式
- **研究内容创新**：发现新现象、探索新领域、揭示新机制
- **研究方法创新**：建立新技术、开发新模型、提出新分析框架

**警示**：避免将"国内首次""填补空白"等无实质内容的套话作为创新点；创新点须有文献对比支撑。

#### 3.2.5 年度计划与预期成果（精简版）

2026版可适当压缩，但仍建议保留简要说明：

```
年度计划（简表形式，无需详细展开）：
  第1年：[主要任务] → [预期阶段性成果]
  第2年：[主要任务] → [预期阶段性成果]
  第3年：[主要任务] → [最终成果输出]

预期成果形式：
  - 学术论文（建议数量、目标期刊层次）
  - 专利（如适用）
  - 数据库/软件（如适用）
  - 人才培养（研究生）
```

### 3.3 第三模块：研究基础

**核心任务**：证明"**申请人有能力完成本研究**"

**篇幅建议**：6~10页

**C类特别说明**：C类重点评估申请人**创新潜力**，研究基础展示的是"我有能力做好这件事"，而非用成果堆砌来证明资历；即使前期成果积累尚浅，也应充分展示与本项目直接相关的初步数据或预研工作。

#### 3.3.1 前期工作基础（核心内容）

```
撰写结构：
  1. 与本项目直接相关的已有工作（重点）
     → 初步实验数据、预研结果
     → 已发表的相关论文（需与本项目研究内容紧密相关）
     → 已建立的关键技术平台/研究体系
  
  2. 代表性成果介绍（5篇代表作为核心）
     → 每篇须说明：本人贡献（具体、可量化）、与本项目的关联性
     → 严禁虚假标注第一作者或通讯作者
     → 严禁篡改作者顺序
```

**5篇代表作选排黄金原则（C类导向）**：

| 维度 | 具体要求 |
|---|---|
| **时效性** | 优先近5年成果；顶刊（CNS正刊/高水平子刊）可适当放宽 |
| **期刊水平** | 至少3篇二区及以上；避开预警期刊和On-hold期刊 |
| **作者顺序** | C类侧重第一作者（贡献权重约70%）；建议≥2篇唯一第一作者二区以上论文 |
| **相关性** | 5篇围绕同一研究方向，形成完整研究链条（机制发现→验证→拓展） |
| **理想配置** | 3篇直接相关 + 1篇拓展研究 + 1篇高影响力旁证 |

**代表作个人贡献声明格式**（2026年强化要求）：
> "本人在该研究中负责：[具体贡献，如：独立设计实验方案、完成全部数据采集与分析、撰写论文初稿及修订]，约占工作量的[X]%。"

#### 3.3.2 工作条件

```
应涵盖：
  - 实验室/计算平台硬件条件（与研究内容匹配的关键设备）
  - 单位支持（启动经费、实验室空间等）
  - 合作资源（如有国内外合作单位，说明合作内容）
  - 数据获取渠道（如涉及大数据、临床数据）
```

精简原则：条件描述须紧扣研究内容，无需罗列所有设备，只需证明"做这件事的条件具备"。

#### 3.3.3 在研项目情况

```
格式：
  项目名称 | 资助来源 | 批准号 | 资助期限 | 本人角色 | 经费 | 与本申请的关系/区别

注意：
  - 需明确说明与本申请项目的区别，避免重复资助嫌疑
  - 若为相关联项目，说明本申请如何在前者基础上深化
```

## 四、个人简历填写规范

```
基本要求：
  - 所有信息必须真实、准确，与证书/证件一致
  - 职称须详细列出各时间段对应职称，严禁跳跃填写
  - 学位信息须与学位证书一致，时间以证书日期为准
  - 博士后人员职称选"无"，不再有"博士后"选项

论文成果填写：
  - 按发表时真实情况如实列出所有作者，不得篡改顺序
  - 署名情况须如实标注（唯一第一/共同第一/唯一通讯/共同通讯/其他）
  - 代表性论文须上传公开发表全文PDF

近五年项目范围界定：
  - 资助期限开始日期晚于2021年1月1日（含）或目前在研的项目
```

## 五、特殊情形处理规范

### 5.1 伦理与科技安全

| 情形 | 处理要求 |
|---|---|
| 涉及人体组织/细胞/器官研究 | 上传伦理委员会审核证明（必须，否则不予资助） |
| 涉及动物实验 | 遵守国家动物伦理与福利规定，部分科学部需提供证明 |
| 涉及高致病性病原体 | 上传生物安全保障承诺书 |
| 涉及人类遗传资源 | 遵守《人类遗传资源管理条例》，提供相关说明 |
| 涉及信息安全/生物信息安全 | 部分科学部要求提供依托单位审查意见 |

### 5.2 AI辅助写作声明规范

```
允许：使用AI辅助检索研究动态、整理参考文献（须人工核实真实性）
禁止：使用AI直接生成申请书；使用未经核实的AI生成内容
要求：必须全面如实声明AI使用情况，按国家规定对相关内容进行标识
```

### 5.3 A类（杰青）附加要求

- 每年在依托单位从事研究工作时间须**9个月以上**
- 选择"临床科学"选项的申请人须额外上传：医师资格证书、执业证书、高级职称资格证书及聘任证书
- 延续资助申请只能选择与第一资助周期相同科学部的申请代码，不得跨科学部

## 六、质量检查清单（提交前逐项核对）

### 形式完整性
- [ ] 申请代码选至二级（4位数字），资助类别、亚类、附注说明填写正确
- [ ] 正文不超过30页
- [ ] 摘要不超过400字，关键词3-5个
- [ ] 所有主要参与者已在线填写简历并上传PDF
- [ ] 代表性论文已上传全文PDF
- [ ] 研究期限填写正确（C/B类3年，A类5年）
- [ ] 所需附件材料（伦理证明等）已按要求上传

### 内容质量
- [ ] 立项依据开篇500字内清晰呈现创新性与科学价值
- [ ] 科学问题聚焦、具体，属于基础研究层面
- [ ] 研究内容有内在逻辑关联，而非平行罗列
- [ ] 创新点有文献对比支撑，避免空泛表述
- [ ] 代表作与本项目高度相关，作者贡献声明具体
- [ ] 前期工作基础包含初步数据或预研结果
- [ ] 全文无涉密、涉政敏感内容

### 诚信合规
- [ ] 所有信息均真实、准确
- [ ] 未使用AI直接生成内容；如使用AI辅助，已声明并核实
- [ ] 不存在超项情况（申请人+主要参与者均已核查）
- [ ] 与在研项目的区别已明确说明

## 七、高频错误与写作禁忌

| 错误类型 | 具体表现 | 改正方向 |
|---|---|---|
| 套话堆砌 | "具有重大理论意义和应用价值""填补国内空白" | 用具体数据、机制、对比说明意义 |
| 方向发散 | 研究内容涵盖5个以上相互独立的子方向 | 聚焦2-3个核心问题，深挖 |
| 目标虚大 | "建立完整理论体系""实现重大突破" | 改为3年内可实现的具体科学目标 |
| 方法缺失 | 只说"做什么"不说"怎么做" | 在研究内容中融入关键方法说明 |
| 基础脱节 | 前期工作与申请课题相关性低 | 挖掘前期工作中与本项目的逻辑联系 |
| 创新虚假 | 创新点无对比文献支撑，仅凭断言 | 每条创新点配"现状+突破"对比说明 |
| 篇幅失控 | 正文超过30页，大量内容重复 | 删除套话，每句话须有实质信息量 |
